Operador REST/SPREAD

Este operador é representado por ...

Ex.:
function fazerAlgumaCoisa( ...coisa ) { }

REST e SPREAD, na verdade, se referem ao mesmo operador.
O que diferencia um do outro é contexto no qual este operador é aplicado.

  • REST é aplicado onde valores isolados são unidos como elementos em um objeto Array.
  • SPREAD é aplicado quando, a partir de um Array, deseja obter-se cada elemento de maneira isolada.
Além disso, este operador geralmente é utilizado no recebimento de parâmetros de funções em JS.
Mas por quê?

Por padrão, funções em JavaScript esperam uma quantidade fixa de parâmetros. Se mais argumentos forem passados na chamada da função além daqueles que já são esperados na assinatura da função, o JavaScript não retorna erro e não interrompe a execução do script. Isso é bom, pois já há nativamente este recurso que garante que uma chamada de função com assinatura diferente da esperada não quebre a aplicação.

Porém, este recurso REST/SPREAD está disponível justamente para trazer um dinamização às assinaturas das funções, abrindo um novo universo de possibilidades e abertura de diversas novas linhas de raciocínio.
Este recurso consiste em recuperar todo e qualquer argumento, independente se era esperado ou não, e passá-los ao escopo da função no formato de um array ordenado, de acordo com a sequência passada. Sendo assim, o parâmetro recebido em array pode ser submetido a estruturas de repetição ou métodos nativos do tipo Array, como o forEach por exemplo.

Esta prática garante que suas funções se tornem muito mais dinâmicas se comparadas aos padrões de assinaturas convencionais.

Aplicação do operador

Assinatura convencional de uma função 

function somarValores(n1, n2){
    return n1 + n2
}

Assinatura utilizando operador REST 

function somarValores(...num){
    let result = 0
    num.forEach(element => result += element)
    return result
}
